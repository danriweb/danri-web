# Архитектура проекта

Проект построен на базе **Next.js 16 (App Router)** и **React 19** с использованием методологии **Feature-Sliced Design (FSD)**.

## Технологический стек

- **Core**: Next.js 16 (App Router), React 19
- **Архитектура**: Feature-Sliced Design (FSD)
- **State Management**: Zustand
- **Стилизация**: Tailwind CSS v4
- **UI Библиотеки**: Shadcn UI (Radix UI), Framer Motion, Lucide React, React Icons
- **Локализация (i18n)**: next-intl
- **Строгая типизация**: TypeScript (`strict: true`)
- **Работа с формами и валидация**: React Hook Form + Zod
- **Линтинг и стандарты**: ESLint, Steiger (FSD linter), Prettier

## Feature-Sliced Design (FSD)

Структура кодовой базы (папка `src/`) строго следует правилам FSD и проверяется линтером **Steiger**.

### Слои (Layers)

1. **`app`** — Инициализация приложения, глобальные стили, провайдеры и настройка маршрутизации App Router. Здесь же обрабатывается i18n на корневом уровне (`[locale]`).
2. **`widgets`** — Крупные, самостоятельные блоки страниц.
3. **`features`** — Пользовательские сценарии и бизнес-ценность (интерактивные элементы).
4. **`entities`** — Бизнес-сущности (данные, типы, небольшие UI компоненты сущностей).
5. **`shared`** — Переиспользуемый код, не привязанный к бизнес-логике.
   - `ui` — UI-кит проекта. Строго разделён на поддиректории:
     - `shadcn` — Базовые, низкоуровневые компоненты из библиотеки Shadcn UI.
     - `constructors` — Более сложные, составные компоненты или кастомные обертки, задающие единый стандарт по проекту.
     - `custom` — Полностью уникальные компоненты.
   - `lib` — Утилиты, хелперы и хуки.
   - `assets` — Статика (шрифты, изображения) и svg-иконки.
   - `model` — Общие типы, сторы и конфиги.

## Управление состоянием (Zustand)

Для глобального стейта используется **Zustand**. Локальное состояние логически изолировано по фичам.
Подробный паттерн описан в [state-management.md](./state-management.md).

## Интернационализация (i18n)

Используется `next-intl` (версия 4.x) в связке с App Router.

- Для навигации (Link, useRouter, usePathname) **всегда** используются хуки и компоненты из `src/i18n/routing.ts`, а не стандартные из `next/navigation` или `next/link`.
- Поддерживаются языки: русский, английский, японский, корейский, немецкий.
- Локализация подгружается с помощью:
  - `getTranslations` на сервере
  - `useTranslations` на клиенте

## UI и Стилизация

- **Tailwind CSS v4** используется для всех стилей проекта, включая сложную адаптивность.
- **Интерактивность:** Широко применяется Framer Motion для декларативных анимаций и скролл-эффектов.
- **Шрифты:** Шрифты подключаются через `@next/font` (Inter, Noto Sans, Funnel Display) и хранятся в CSS переменных, которые интегрированы с Tailwind.

Подробные правила работы с хуками компонентов и UI-обертками вынесены в [components.md](./components.md).

## Проверка качества кода (Linting)

Проект использует многоуровневый линтинг:

- **ESLint** — базовый TypeScript/React линтинг.
- **Steiger** (`npm run steiger`) — следит за тем, чтобы не нарушались архитектурные слои FSD (imports rules, public api).
- **Prettier** — занимается форматированием и сортировкой импортов.
