# Правила и создание компонентов

В этом документе описаны правила стилизации, именования и общие принципы создания React компонентов в проекте.

## 1. Базовые компоненты

В качестве базы для UI-кита используется библиотека компоненты **Shadcn UI**.
Поверх них или с нуля собираются консистентные `Standard*` обертки (например, `StandardSheet`, `StandardAdaptiveOverlay`), которые задают унифицированное поведение по всему проекту.

## 2. Именование компонентов

- **Формат:** `PascalCase`.
- Компонент и файл, в котором он находится, должны называться одинаково.
- **Примеры файлов:** `ContactModal.tsx`, `StandardAdaptiveOverlay.tsx`.

## 3. Написание React компонентов

- **Экспорты:** Используются только именованные экспорты (`export const Component = () => {}`). Исключение — системные файлы Next.js (`page.tsx`, `layout.tsx`), где нужен `export default`.
- **Компоненты:** Создаются исключительно через стрелочные функции.
- **Пропсы:** Интерфейсы свойств называются так же, как компонент, плюс суффикс `Props` (например, `ContactModalProps`).
